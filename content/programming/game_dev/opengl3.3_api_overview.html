1. Introduction to OpenGL

OpenGL (Open Graphics Library) is a cross-platform API for rendering 2D and 3D vector graphics. The API is used to interact with a GPU, allowing developers to create graphics programs, primarily for real-time rendering in games, simulations, and other visualizations.

2. Context Creation

OpenGL requires a context to function, which is typically created by a windowing library such as GLFW, SDL, or native platform APIs. OpenGL context holds the state that affects rendering and is necessary for OpenGL commands to work.

3. OpenGL Shading Language (GLSL)

3.1 Shaders

OpenGL uses shaders written in GLSL for programmable rendering. The main types of shaders are:

  • Vertex Shader: Processes vertex data and applies transformations.
  • Fragment Shader: Computes the color and other attributes for each fragment (pixel).

3.2 Shader Compilation

Shaders are written in GLSL and must be compiled and linked into a program object:

  • glCreateShader: Creates a new shader object.
  • glShaderSource: Loads the shader source code.
  • glCompileShader: Compiles the shader source.
  • glCreateProgram: Creates a new program object.
  • glAttachShader: Attaches a compiled shader to the program.
  • glLinkProgram: Links all attached shaders into an executable.
  • glUseProgram: Activates the shader program.

4. Buffers

4.1 Vertex Buffer Object (VBO)

A VBO stores vertex data in GPU memory:

  • glGenBuffers: Generates buffer object names.
  • glBindBuffer: Binds a buffer to a target (e.g., GL_ARRAY_BUFFER).
  • glBufferData: Creates and initializes buffer object's data store.

4.2 Element Buffer Object (EBO)

An EBO stores indices for drawing with vertex array objects:

  • glBindBuffer(GL_ELEMENT_ARRAY_BUFFER, ...): Binds the EBO to the array buffer.

4.3 Vertex Array Object (VAO)

VAOs store state related to vertex attribute configuration:

  • glGenVertexArrays: Generates VAO object names.
  • glBindVertexArray: Binds the VAO for use.
  • glVertexAttribPointer: Specifies the layout of the vertex data.
  • glEnableVertexAttribArray: Enables a generic vertex attribute array.

5. Texturing

5.1 Texture Creation

Textures are used to map images onto geometric surfaces:

  • glGenTextures: Generates texture object names.
  • glBindTexture: Binds a texture to a target (e.g., GL_TEXTURE_2D).
  • glTexImage2D: Specifies a 2D texture image.
  • glTexParameteri: Sets texture parameters (e.g., wrapping, filtering).

5.2 Texture Units

OpenGL allows multiple textures to be bound at once, each to a different texture unit:

  • glActiveTexture: Specifies the active texture unit.

6. Drawing

6.1 Primitive Drawing

OpenGL supports various drawing primitives such as points, lines, and triangles:

  • glDrawArrays: Renders primitives from array data.
  • glDrawElements: Renders primitives using indices.

7. Framebuffers

7.1 Default Framebuffer

By default, OpenGL renders to a window provided by the OS.

7.2 Framebuffer Object (FBO)

FBOs allow off-screen rendering for post-processing, shadow maps, etc.:

  • glGenFramebuffers: Generates FBO object names.
  • glBindFramebuffer: Binds the FBO.
  • glFramebufferTexture2D: Attaches a texture to the FBO.
  • glCheckFramebufferStatus: Checks if the FBO is complete.

8. State Management

8.1 Enabling and Disabling Capabilities

OpenGL has many state variables that control rendering behavior:

  • glEnable: Enables a feature (e.g., GL_DEPTH_TEST, GL_BLEND).
  • glDisable: Disables a feature.
  • glDepthFunc: Specifies the depth comparison function.
  • glBlendFunc: Specifies the blending function.

9. Cleanup

Once OpenGL objects (like buffers, shaders, textures, and programs) are no longer needed, they should be deleted:

  • glDeleteBuffers: Deletes buffer objects.
  • glDeleteTextures: Deletes texture objects.
  • glDeleteProgram: Deletes shader programs.
  • glDeleteShader: Deletes shader objects.
  • glDeleteVertexArrays: Deletes VAO objects.

10. Error Handling

OpenGL provides error checking functions:

  • glGetError: Returns error codes (e.g., GL_NO_ERROR, GL_INVALID_ENUM).

11. OpenGL Versions and Extensions

OpenGL versions introduce new features, and extensions allow additional functionalities beyond the core API:

  • glGetString: Queries version, vendor, and extension information.
  • glGetIntegerv: Retrieves specific OpenGL state values (e.g., maximum texture size).
